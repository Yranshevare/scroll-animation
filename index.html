Animation based on scroll using CSS

Scroll animations in CSS bring elements to life as they enter the viewport, making pages more interactive. By using CSS @keyframes you can trigger animations when elements appear on screen helps control the animation start, stop, and delay based on scroll position, allowing for precise animation timing and effects as users scroll through the page.

Introduction


animation-timeline:

Feature that allows developers to synchronize animations with specific events , like scroll position. Animation-timeline can dynamically adjust the animation progress based on an external trigger. This enables more flexible and responsive animations, particularly useful for scroll-triggered effects or custom interactive elements.

animation-range:

is designed to control the specific portion or "range" of an animation that plays based on external factors like scroll position allowing for more control over animation when animations start, pause, or progress.

animation-timeline

scroll(): animation start to proceed once you start scrolling and execute completely once you scroll through your whole web page

(NOTE: scroll():- default i.e, scroll along y axis, scroll(x):- scroll along x-axis, scroll(y):-scroll along y axis)


view(): animation proceed as soon as element enters into the viewport and complete at the end of viewport

(NOTE: view() default full viewport, view(top bottom) the top and bottom of the viewport, view(bottom) bottom of the viewport)

animation-range

Unlike traditional animations that run continuously, animation-range enables finer control, so only a chosen part of the animation plays within a specified range of user input or viewport area.

animation-range-start: decide when to start animation

animation-range-end: decide when to end animation

animation-range: start end
